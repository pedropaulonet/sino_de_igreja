🔧 Sino de Igreja - How To Técnico

Documentação técnica do programa. Feito para quem precisar mexer depois.

🛠️ Stack

  • Linguagem: Python 3
  • Interface: curses (biblioteca padrão do Linux)
  • Áudio: pygame (preferencial) ou fallback: ffplay, paplay, aplay, omxplayer
  • Config: JSON
  • Plataforma: Linux, Raspberry Pi (ARM)

🏗️ Arquitetura

O programa é baseado em 3 classes principais:

AudioPlayer

Cuida de tocar e parar os áudios. Detecta automaticamente qual player está disponível no sistema.

  • __init__() - Procura players: pygame → ffplay → paplay → aplay → omxplayer
  • play(filepath, blocking) - Toca um arquivo. blocking=True espera terminar.
  • stop() - Para o player atual (cada um tem seu comando de kill)

ScheduleManager

Gerencia horários e toques. O "cérebro" do programa.

  • carregar_config() - Lê o config.json
  • hora_permitida() - Retorna sempre True. Função mantida para possível uso futuro.
  • obter_proximos_toques(limite) - Lista os próximos horários ordenados
  • tocar_sino(item) - Toca o sino X vezes. Tem flag parar_loop para parar quando [S] é pressionado
  • parar_audio() - Para o som E o loop de repetições
  • verificar_e_tocar() - Called a cada minuto. Verifica se é hora de tocar e executa

InterfaceCurses

Interface visual do terminal. Usa caracteres de controle para desenhar na tela.

  • iniciar() - Inicia o curses com wrapper
  • _desenhar(stdscr) - Desenha a UI. Só redesenha quando segundo muda (evita flicker)
  • _loop_principal(stdscr) - Loop principal. Lê teclas e atualiza a cada 0.1s

📋 Fluxo de Execução

main()
  ├── carrega config.json
  ├── se --console → modo_console()
  └── senão → InterfaceCurses.iniciar()
        └── curses.wrapper(_loop_principal)
              ├── loop a cada 0.1s
              │   ├── lê tecla
              │   │   ├── Q → sai
              │   │   ├── R → recarrega config
              │   │   ├── T → toca sino 1x
              │   │   ├── S → para áudio e loop
              │   │   ├── ESPAÇO → toca sino 3x
              │   │   └── 1-9 → toca horário específico
              │   ├── verifica minuto → verificar_e_tocar()
              │   └── redesenha se segundo mudou
              └── _desenhar() → mostra relógio, lista de próximos toques, rodapé

📁 config.json

{
  "sons": {            # Mapeamento nome → arquivo
    "sino": "sounds/sino.mp3",
    "nossa_senhora": "sounds/nossa_senhora.mp3"
  },
  "programacao": [     # Lista de toques
    {
      "hora": 7,       # 0-23
      "minuto": 0,     # 0-59
      "som": "sino",   # chave do mapa "sons"
      "repeticoes": 7  # quantas vezes toca
    }
  ]
}

🐛 Debugging

  • LSP errors no VS Code: Falsos positivos. pygame e curses são resolvidos em runtime. O código funciona.
  • Sem áudio: Verifique se o arquivo MP3 existe, se não está corrompido, e se o player tá instalado
  • Não sai com Q: Verifica se o terminal suporta curses (executar em terminal real)

✏️ Adicionar novo som

  1. Coloque o MP3 em sounds/
  2. Adicione no config.json: 
    {"hora": 12, "minuto": 0, "som": "novo_som", "repeticoes": 3}
  3. Adicione o mapeamento em "sons": 
    "novo_som": "sounds/novo_som.mp3"
  4. Pressione R para recarregar

🔧 Modificar horários

Edite direto no config.json. Por exemplo, para tocar às 15h:

{"hora": 15, "minuto": 0, "som": "sino", "repeticoes": 3}
